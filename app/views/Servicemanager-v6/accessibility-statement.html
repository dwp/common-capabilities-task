Accessibility statement for DWP Task

This accessibility statement applies to the DWP Task application

This application is run by the DWP Task team. We want as many people as possible to be able to use this application. For example, that means you should be able to:

  • change colours, contrast levels and fonts using browser or device settings
  • Update a customer's bank details
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using a keyboard or speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this application is

We know some parts of this application are not fully accessible:

  • content relationships are not always communicated
  • links do not always have accessible text
  • instructional content for user input is sometimes missing or incorrect
  • some content has not been coded, for example empty state content
  • focus is not always set to the status message, so status messages are not always announced after you perform an action
  • our text is not as simple as possible to understand

Feedback and contact information

If you find any problems not listed on this page or think we’re not meeting accessibility requirements, contact the DWP Task team by raising an issue in DWP Place.

If you need information on this application in a different format like accessible PDF, large print, easy read, audio recording or braille:

  • email DWPTask@dwp.gov.uk

We’ll consider your request and get back to you in 3 days.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website's accessibility

DWP Task is committed to making its application accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

The application has been tested against the Web Content Accessibility Guidelines (WCAG) 2.2 AA standard.

This website is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to the non-compliances listed below.

The content listed below is non-accessible for the following reasons.

Non-accessible content

Non-compliance with the accessibility regulations

Some of the content relationships are not communicated in code, so people using a screen reader cannot easily understand how one content type relates to another. This fails WCAG 2.2 success criterion 1.3.1 (Info and Relationships)

Some links do not have visually hidden text, so people using a screen reader will not be able to understand the purpose of the link. This fails WCAG success criterion 2.4.4 (Link purpose).

Some content does not have instructions or guidance to help people understand what to do at user input. Some instructional content is incorrect. This fails WCAG 2.2 success criterion 3.3.3 (Labels or instructions).

Some of our content is not programmatically determined. Where content is missing, people using a screen reader cannot easily understand all of the content. This fails WCAG 2.2 success criterion 4.1.2 (Name Role value).

Some of our status messages are not announced, so people using a screen reader cannot easily understand what has happened following their action. This fails WCAG 2.2 success criterion 4.1.3 (Status messages).

We plan to implement all fixes by March 2025.

Content that’s not within the scope of the accessibility regulations

Some of our text is not as simple as possible to understand, which impacts readability and the ability of the user to understand content. Some content has been coded in kebab case, lower case connected by hyphens. Some screen reader users, for example those using JAWS, will be affected, as every word and hyphen will be read out loud. WCAG 2.2 does not provide any guidance for this kind of content.

We plan to implement a solution for content to appear normally to users by March 2025.
